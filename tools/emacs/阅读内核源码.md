## 总体思路

阅读 Linux 内核代码时，`imenu-list` 适合看当前文件结构，`TAGS` 适合跳定义，`cscope` 更适合查调用关系、符号引用和文本搜索。

建议工作流：

1. 在内核源码根目录生成 `TAGS` 和 `cscope.out`。
2. Emacs 打开源码文件后，先用 `F8` 看当前文件函数列表。
3. 用 `M-.` 跳到函数、宏、变量定义。
4. 用 `M-,` 返回。
5. 用 `C-c s ...` 做 `cscope` 查询，比如查调用者、全局定义、文本出现位置。


## 前置准备

在内核源码根目录执行：

```bash
make TAGS
make cscope
```

生成后通常会得到：

- `TAGS`
- `cscope.out`


## Emacs 配置效果

当前 `init.el` 已经加入以下能力：

- `C` 文件里使用 Linux 风格缩进
- 自动开启代码折叠 `hs-minor-mode`
- 菜单栏里加入 `imenu` 索引
- `M-.` / `M-,` / `M-?` 用于 `xref`
- `C-c t` 自动加载当前项目最近的 `TAGS`
- `C-c s ...` 使用 `cscope` 查询


## 当前文件结构

| 快捷键 | 说明 |
|--------|------|
| `F8` | 打开并聚焦 `imenu-list` |
| `TAB` | 折叠或展开当前节点 |
| `q` | 关闭 `imenu-list` 窗口 |

说明：

- `imenu-list` 对 C 文件通常更擅长显示函数
- 全局变量、宏定义、调用关系更建议交给 `xref` 和 `cscope`


## TAGS 与 xref

先在内核任意源码文件里执行一次：

| 快捷键 | 说明 |
|--------|------|
| `C-c t` | 自动加载最近的 `TAGS` 文件 |

常用跳转：

| 快捷键 | 说明 |
|--------|------|
| `M-.` | 跳到光标处符号定义 |
| `M-,` | 返回上一个位置 |
| `M-?` | 查找引用 |

适合：

- 查看函数定义
- 查看宏定义
- 查看全局变量定义


## cscope 查询

在已经生成 `cscope.out` 的源码树里，可以直接用下面这些键：

| 快捷键 | 说明 |
|--------|------|
| `C-c s d` | 查找全局定义 |
| `C-c s s` | 查找符号出现位置 |
| `C-c s c` | 查找谁调用了该函数 |
| `C-c s t` | 按文本搜索 |

建议用法：

1. 光标放到函数名上。
2. `C-c s d` 看定义。
3. `C-c s c` 看调用者。
4. `M-,` 返回。

`cscope` 结果会通过 Emacs 的 `xref` 列表显示，可以回车跳转。


## 阅读内核时的推荐操作

| 快捷键 | 说明 |
|--------|------|
| `F8` | 看当前文件函数结构 |
| `F9` | 在右侧打开当前目录 |
| `M-.` | 跳定义 |
| `M-,` | 返回 |
| `C-c s d` | 查定义 |
| `C-c s c` | 查调用者 |
| `C-c s t` | 查文本 |

推荐布局：

1. 左边放源码。
2. 右边用 `F8` 放 `imenu-list` 或 `F9` 放 `dired`。
3. 需要查关系时用 `M-.` 或 `C-c s ...`。


## 什么时候用哪种方式

| 场景 | 推荐方式 |
|------|----------|
| 看当前文件有哪些函数 | `imenu-list` |
| 跳到当前符号定义 | `M-.` |
| 返回上一个位置 | `M-,` |
| 查谁调用了这个函数 | `C-c s c` |
| 查某个宏或文本出现在哪 | `C-c s t` 或 `C-c s s` |
| 浏览当前目录文件 | `F9` |
