# 打字练习

Emacs 自带的功能更偏编辑器本身，如果想在 Emacs 里做类似在线打字测试的练习，推荐安装 `speed-type` 包。它可以随机给出文本，输入时高亮正确/错误字符，结束后显示用时、字符数和 WPM。

## 安装

先确保 Emacs 已经可以使用 MELPA。编辑 `~/.emacs.d/init.el`，加入：

```elisp
(require 'package)
(add-to-list 'package-archives
             '("melpa" . "https://melpa.org/packages/") t)
(package-initialize)
```

重启 Emacs 后执行：

```text
M-x package-refresh-contents
M-x package-install RET speed-type RET
```

如果使用 `use-package` 管理包，也可以写成：

```elisp
(use-package speed-type
  :ensure t
  :commands (speed-type-text
             speed-type-region
             speed-type-buffer))
```

## 基本使用

| 命令 | 说明 |
|------|------|
| `M-x speed-type-text` | 随机选择一段文本开始练习 |
| `M-x speed-type-region` | 用当前选中的文本做练习材料 |
| `M-x speed-type-buffer` | 用整个当前 buffer 做练习材料 |

常用流程：

1. 打开 Emacs。
2. 执行 `M-x speed-type-text`。
3. 看到练习文本后直接开始输入。
4. 输入过程中注意高亮提示，输错就用正常编辑键修正。
5. 全部输入完成后查看统计结果，比如 WPM、耗时和字符数。

## 使用自己的材料练习

练习英文、代码、命令行或自己的笔记时，建议直接用已有 buffer/region：

1. 打开一段想练习的文本，比如 Markdown、代码或命令示例。
2. 选中一小段文本。
3. 执行 `M-x speed-type-region`。
4. 按提示输入，结束后查看统计。

如果想练代码输入，可以打开代码文件后执行：

```text
M-x speed-type-buffer
```

这样比随机英文更贴近日常开发输入习惯。

## 建议配置

可以给常用命令绑定快捷键：

```elisp
(global-set-key (kbd "C-c t t") #'speed-type-text)
(global-set-key (kbd "C-c t r") #'speed-type-region)
(global-set-key (kbd "C-c t b") #'speed-type-buffer)
```

如果第一次运行随机文本时下载 Project Gutenberg 资料较慢，可以先用 `speed-type-region` 练本地文本，等网络正常时再使用 `speed-type-text`。

## 练习建议

- 每次练 5 到 10 分钟，比长时间硬练更容易坚持。
- 优先保证准确率，再逐步提高速度。
- 多用 `speed-type-region` 练自己的常用输入内容，比如 shell 命令、代码片段、英文文档。
- 如果某些组合键经常出错，把相关单词或代码片段单独保存成练习材料反复练。

## 可选方案

如果想尝试其它风格的打字练习，也可以搜索并安装：

| 包 | 特点 |
|----|------|
| `typit` | 类似 10fastfingers 的限时单词练习 |
| `monkeytype` | 类似 monkeytype.com 的 Emacs 内练习体验 |

安装方式与 `speed-type` 类似：

```text
M-x package-install RET typit RET
M-x package-install RET monkeytype RET
```
